<h2 style="color:#0070c0; border-bottom:2px solid #0070c0; padding-bottom:5px; margin-top:30px;">Toolpath Display and Filter Application</h2><p>Pause the program. Now you can click anywhere on the toolpath, and it will jump to that position. Why can you click anywhere? Because our <strong style="color:#e46c0a;">filter</strong> is set to 'Toolpath'.</p>

<p>But pay attention: if your filter is set to another type, like 'Drafting Filter', you won't be able to click on the toolpath. Only when the <strong style="color:#e46c0a;">filter is 'Toolpath' or 'No Selection Filter'</strong> will you be able to click on it.</p>

<h3 style="color:#3399ff; border-left:5px solid #ff9900; padding-left:10px; margin:25px 0;">Three Highlighted Key Points</h3><p>Also, these three areas, everyone must pay attention: <strong style="color:#e46c0a;">they must be highlighted</strong>. If the middle one isn't highlighted, you won't be able to click the toolpath; if the two at the back aren't highlighted, your rapid move lines or the entire toolpath will disappear, and you won't see them at all. Usually, all three are highlighted, which ensures you can view and operate the toolpath normally.</p>
<h2 style="color:#0070c0; border-bottom:2px solid #0070c0; padding-bottom:5px; margin-top:30px;">Tool Axis Explained</h2><p>Let's double-click to open and look at the 'Tool Axis' below.</p>
<h3 style="color:#3399ff; border-left:5px solid #ff9900; padding-left:10px; margin:25px 0;">Default Tool Axis: Perpendicular to First Face</h3><p>Currently, the tool axis here is 'Axis perpendicular to first face'. Why? Because for operations like <strong style="color:#e46c0a;">Floor Wall Milling</strong>, its default tool axis is perpendicular to the first face. Usually, we don't need to change it.</p>
<h3 style="color:#3399ff; border-left:5px solid #ff9900; padding-left:10px; margin:25px 0;">Common Tool Axis: +ZM Axis</h3><p>Generally, during normal machining, it's mostly the <strong style="color:#e46c0a;">+ZM Axis</strong>. Except for Floor Wall Milling, most other commands, ninety-nine percent, use the +ZM axis.</p>
<p>The meaning of this is that our tool axis is <strong style="color:#e46c0a;">upwards</strong>, meaning the Z-axis is upwards, machining from top to bottom. This is how +ZM axis machining works.</p>
<p>When learning 3-axis machining, it's basically all about the +ZM axis. Almost all programs are like this. However, for special cases like Floor Wall Milling, setting it perpendicular to the first face is also acceptable.</p>

<h3 style="color:#3399ff; border-left:5px solid #ff9900; padding-left:10px; margin:25px 0;">When to Modify Tool Axis</h3><p>Everyone should now understand the meaning of the tool axis. We mainly need to change it when learning <strong style="color:#e46c0a;">4-axis or 5-axis simultaneous machining</strong>. For 3-axis machining, we generally don't need to adjust it much.</p>
<h2 style="color:#0070c0; border-bottom:2px solid #0070c0; padding-bottom:5px; margin-top:30px;">Cut Region Space Range: Bottom Face</h2><p>Let's look further down at 'Cut Region Space Range' and 'Bottom Face'.</p>
<p>Looking at this diagram, there's 'Bottom Face' and 'B'. I personally think this 'B' method is used quite rarely. Because when we later learn <strong style="color:#e46c0a;">3D machining</strong>, we can directly machine sloped surfaces like this. This 'B' is specifically for machining sloped surfaces.</p>
<p>Floor Wall Milling is typically for 2D machining. While it can occasionally machine 3D (sloped surfaces), I don't think the results are particularly good. So, I don't really recommend using this function. Everyone just needs to know that such a function exists.</p>
<p>Typically, we will choose <strong style="color:#e46c0a;">Bottom Face</strong>. This way, it directly machines up to this edge, and sloped areas are not machined.</p>
<h2 style="color:#0070c0; border-bottom:2px solid #0070c0; padding-bottom:5px; margin-top:30px;">Cutting Method Explained</h2><p>Moving on, let's look at our 'Cutting Method'. Currently, the default is 'Follow Perimeter'.</p>

<div align="center"><img src="https://raw.githubusercontent.com/cncabc-sketch/Siemens-NX-CAM-Technical-Guides/main/Articles/IMG/%E7%AC%AC9%E8%8A%82%E8%AF%BE%20%20%E4%B8%89%E8%BD%B4/tool-axis-and-cutting-method-04.webp" style="max-width:100%; border-radius:10px;"></div>

<p>What does 'Cutting Method' mean? Simply put, it's <strong style="color:#e46c0a;">the way the toolpath moves</strong>. Let's change it to 'Follow Part' and see if there's any change. For our simple face milling, there's actually no change.</p>
<p>However, if we change it to <strong style="color:#e46c0a;">Contour</strong>, then there will definitely be a change. 'Contour' mode only machines contours. Since we are currently face milling, it's not applicable, and it will give an error: 'This component cannot perform face cutting on a planar surface'. So, face milling definitely cannot use 'Contour' mode.</p>
<h3 style="color:#3399ff; border-left:5px solid #ff9900; padding-left:10px; margin:25px 0;">One-Way Cutting</h3><p>Let's try <strong style="color:#e46c0a;">One-Way</strong>. One-way is definitely possible. See? It engages the tool from this side, moves to that side, then lifts the tool and returns, then engages the tool from that side and moves back. This is a one-way machining method: move across, lift tool and return, move across again, lift tool and return again.</p>
<h3 style="color:#3399ff; border-left:5px solid #ff9900; padding-left:10px; margin:25px 0;">Zig-Zag Cutting</h3><p>Since you understand one-way, <strong style="color:#e46c0a;">Zig-Zag</strong> is even easier to grasp. It just moves across, then directly down, then across again, then down again. That is: move across, go down, return, then move across again, go down, return. It just keeps milling like that.</p>
<p>This, then, is our 'Cutting Method'.</p>
<h3 style="color:#3399ff; margin-top:20px;">Summary: Pitfall Guide</h3><p>Everyone must pay attention to the filter settings, <strong style="color:#e46c0a;">especially when you're first practicing; not being able to click toolpaths is often because the filter isn't selected correctly</strong>. Furthermore, the tool axis usually doesn't need to be changed in 3-axis machining, mainly focus on the +ZM axis. The choice of cutting method depends on the type of machining; for example, face milling usually selects 'One-Way' or 'Zig-Zag', while 'Contour' mode is not suitable for planar cutting. Understanding these will greatly improve your programming efficiency and machining stability.</p>
