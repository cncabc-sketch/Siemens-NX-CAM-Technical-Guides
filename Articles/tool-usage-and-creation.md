<br><h2 style="color:#0070c0; border-bottom:2px solid #0070c0; padding-bottom:5px; margin-top:30px;">Previous Review and Function Overview</h2><br><p>Hello everyone, I'm Master Wang. Let's pick up where we left off. We've already covered all the functions in that first section. As for "<strong style="color:#e46c0a; font-weight:bold;">Trim Boundary</strong>" and "<strong style="color:#e46c0a; font-weight:bold;">Specify Boundary</strong>," these aren't used very often, so we'll just briefly touch upon them for now and might revisit them later.</p>
<br><p><strong style="color:#e46c0a;">Listen up!</strong> Pay special attention to the "<strong style="color:#e46c0a; font-weight:bold;">Specify Bottom Boundary</strong>" function. Frankly, I don't particularly recommend using it, but since it's in the software, we still need to know how to use it to avoid potential pitfalls later on.</p>
<br><h2 style="color:#0070c0; border-bottom:2px solid #0070c0; padding-bottom:5px; margin-top:30px;">Core: Tool Selection and Generation</h2><br><h3 style="color:#3399ff; border-left:5px solid #ff9900; padding-left:10px; margin:25px 0;">Tool Selection</h3><br><p>Look at this section, the "<strong style="color:#e46c0a; font-weight:bold;">Tool</strong>" option. It's straightforward: choose the tool you want to use. For example, if the default is a D16 (16 mm diameter) tool, we can switch to a D4.7 R0 tool (4.7 mm diameter, 0 corner radius).</p>
<br><p><strong style="color:#e46c0a;">Remember these three crucial steps:</strong><br><br>1. <strong style="color:#e46c0a;">Specify Blank</strong><br><br>2. <strong style="color:#e46c0a;">Specify Cut Area</strong><br><br>3. <strong style="color:#e46c0a;">Select Tool</strong></p>
<br><p>As long as you get these three steps right, you generally won't make major mistakes. Don't just rely on software simulations; you need to observe the actual cutting sparks and results!</p>

<br><h3 style="color:#3399ff; border-left:5px solid #ff9900; padding-left:10px; margin:25px 0;">Importance of Program Generation</h3><br><p>After selecting the tool and setting the parameters, the most important step is to click the "<strong style="color:#e46c0a; font-weight:bold;">Generate</strong>" button. Once you click it, the program will generate the toolpath for you.</p>
<br><p>Let me give you an example. Say you change the tool from D4.7 to D5.8. After making the change, the tool in your toolpath might still be using the D4.7 parameters internally. You <strong style="color:#e46c0a;">must</strong> click "<strong style="color:#e46c0a; font-weight:bold;">Generate</strong>" for your changes to be truly applied.</p>
<br><p>This "Generate" step is the <strong style="color:#e46c0a;">most critical</strong> one every time you modify parameters. You can either change all parameters at once and then generate, or change one step and generate. Either way is fine. But any change requires generation!</p>
<br><p>We'll talk about "<strong style="color:#e46c0a; font-weight:bold;">Replay</strong>," "<strong style="color:#e46c0a; font-weight:bold;">Verify</strong>," and "<strong style="color:#e46c0a; font-weight:bold;">List</strong>" later. But the importance of "Generate" outweighs them all.</p>
<br><h2 style="color:#0070c0; border-bottom:2px solid #0070c0; padding-bottom:5px; margin-top:30px;">Methods for Tool Creation</h2><br><h3 style="color:#3399ff; border-left:5px solid #ff9900; padding-left:10px; margin:25px 0;">Method One: Copy and Paste</h3><br><p>If the current tool library doesn't have the tool you need, like a D17 tool, don't rush to search for it. We can "<strong style="color:#e46c0a; font-weight:bold;">Copy</strong>" an existing tool and then "<strong style="color:#e46c0a; font-weight:bold;">Paste</strong>" it.</p>

<div align="center"><img src="https://raw.githubusercontent.com/cncabc-sketch/Siemens-NX-CAM-Technical-Guides/main/Articles/IMG/%E7%AC%AC8%E8%8A%82%E8%AF%BE%20%20%E4%B8%89%E8%BD%B4/tool-usage-and-creation-02.webp" style="max-width:100%; border-radius:10px;"></div>

<br><p>Next, double-click the pasted tool, rename it to "D17," and confirm. Just like that, a new D17 tool appears in your tool library.</p>
<br><h3 style="color:#3399ff; border-left:5px solid #ff9900; padding-left:10px; margin:25px 0;">Method Two: New Tool Creation</h3><br><p>Besides copying and pasting, we can also directly click "<strong style="color:#e46c0a; font-weight:bold;">New</strong>" to create a tool. For example, let's create a D10 R3.1 tool.</p>
<br><p><strong style="color:#e46c0a;">Pay attention:</strong> If you try to create a D10 R3 tool, and your tool library already has one with that name, you won't be able to create another tool with the same name. So, it's best to use a distinct name, such as adding a prefix like "E10 R3.1" to avoid conflicts.</p>
<br><p>Simply enter the name and corresponding parameters, such as diameter D10 and corner radius R3.1, then confirm. The new tool is now created. This is effectively the same as copying, pasting, and renaming. Both methods work, use whichever you find more convenient.</p>
<br><h3 style="color:#3399ff; border-left:5px solid #ff9900; padding-left:10px; margin:25px 0;">Method Three: Create within an Operation</h3><br><p>When creating an operation, there will also be a "<strong style="color:#e46c0a; font-weight:bold;">Create Tool</strong>" option. This function is <strong style="color:#e46c0a;">identical</strong> to the "New Tool" option mentioned earlier. You only need to learn one tool creation method; there's no need to master every single one.</p>

<div align="center"><img src="https://raw.githubusercontent.com/cncabc-sketch/Siemens-NX-CAM-Technical-Guides/main/Articles/IMG/%E7%AC%AC8%E8%8A%82%E8%AF%BE%20%20%E4%B8%89%E8%BD%B4/tool-usage-and-creation-03.webp" style="max-width:100%; border-radius:10px;"></div>

<br><p>So, in summary, you can create tools either externally via "New" or "Copy and Paste," or directly within an operation. All methods are viable.</p>
<br><h2 style="color:#0070c0; border-bottom:2px solid #0070c0; padding-bottom:5px; margin-top:30px;">Modifying Tool Parameters and Output Settings</h2><br><h3 style="color:#3399ff; border-left:5px solid #ff9900; padding-left:10px; margin:25px 0;">Editing Tool Parameters</h3><br><p>Click "<strong style="color:#e46c0a; font-weight:bold;">Edit Display</strong>" to modify tool parameters. However, there's a pitfall here: if you directly change the name or diameter, for example, changing D10 R1 to D11 R1, the tool's name in the list will still show D10 R1, but its internal parameters will have changed. This is highly confusing, and <strong style="color:#e46c0a;">I do not recommend doing this!</strong></p>
<br><p>What I suggest changing here are only the <strong style="color:#e46c0a;">physical dimensions</strong> of the tool, such as "<strong style="color:#e46c0a; font-weight:bold;">Tool Length</strong>" and "<strong style="color:#e46c0a; font-weight:bold;">Cutting Length</strong>." For instance, changing the tool length to <strong style="color:#e46c0a;">150mm</strong> and the cutting length to <strong style="color:#e46c0a;">50mm</strong> is perfectly fine. As for the "<strong style="color:#e46c0a; font-weight:bold;">Number of Flutes</strong>," you can change this, but usually, it doesn't have a significant impact if left as default; the software's default is often sufficient.</p>
<br><p>So, after modifying these physical dimensions, the tool's name in the list will remain the original D12, but its actual dimensions will have changed according to your input.</p>
<br><h3 style="color:#3399ff; border-left:5px solid #ff9900; padding-left:10px; margin:25px 0;">Output Settings and Post-Processing</h3><br><p>Next, let's look at the "<strong style="color:#e46c0a; font-weight:bold;">Output</strong>" options. Here we have "<strong style="color:#e46c0a; font-weight:bold;">Tool Number</strong>," "<strong style="color:#e46c0a; font-weight:bold;">Compensation</strong>," and "<strong style="color:#e46c0a; font-weight:bold;">Length Compensation</strong>." By default, they are all set to 0.</p>

<div align="center"><img src="https://raw.githubusercontent.com/cncabc-sketch/Siemens-NX-CAM-Technical-Guides/main/Articles/IMG/%E7%AC%AC8%E8%8A%82%E8%AF%BE%20%20%E4%B8%89%E8%BD%B4/tool-usage-and-creation-04.webp" style="max-width:100%; border-radius:10px;"></div>

<br><p>What do these mean? Simply put, they relate to the G-code generated during <strong style="color:#e46c0a;">post-processing</strong>. For example, your G-code might contain <strong style="color:#e46c0a;">M6 T1</strong> (tool change to tool 1) or <strong style="color:#e46c0a;">G43 H1</strong> (length compensation 1).</p>
<br><p>Your "<strong style="color:#e46c0a; font-weight:bold;">Tool Number</strong>" corresponds to that T-number. If you set it to <strong style="color:#e46c0a;">1</strong> here, then in the post-processed program, this tool number will be <strong style="color:#e46c0a;">T1</strong>. If your company doesn't have strict requirements, or if your workshop, like ours, uses non-standardized tools where operators manually perform tool offsetting and input compensation, then setting this to <strong style="color:#e46c0a;">0</strong> is also acceptable; the operator will fill it in manually.</p>
<br><p>However, if you need the program to automatically output <strong style="color:#e46c0a;">T1 H1</strong>, then set all three parameters (Tool Number, Compensation, Length Compensation) to <strong style="color:#e46c0a;">1</strong>, then "<strong style="color:#e46c0a; font-weight:bold;">Generate</strong>."</p>
<br><p><strong style="color:#e46c0a;">Listen up!</strong> Changing these parameters here won't show any visual change in the toolpath within the software. Where do you see the effect? Only when you <strong style="color:#e46c0a;">post-process</strong> the G-code! For example, if you select post-processing and then view the generated NC file, you'll find codes like <strong style="color:#e46c0a;">M6 T1</strong> or <strong style="color:#e46c0a;">G43 H1</strong> inside.</p>
<br><br><h3 style="color:#3399ff; margin-top:20px;">Summary: Pitfall Guide</h3><br><ul><br>    <li><strong style="color:#e46c0a; font-weight:bold;">Always "Generate":</strong> Any modification to tool parameters (diameter, length, compensation, etc.) <strong style="color:#e46c0a;">must</strong> be followed by clicking "<strong style="color:#e46c0a; font-weight:bold;">Generate</strong>" to take effect. Otherwise, your program will still be using the old parameters. This is the most common mistake, so remember it!</li><br>    <li><strong style="color:#e46c0a; font-weight:bold;">Be Cautious with "Edit Display":</strong> Do not use "<strong style="color:#e46c0a; font-weight:bold;">Edit Display</strong>" to change the tool's "<strong style="color:#e46c0a; font-weight:bold;">name</strong>" or core identifying parameters like "<strong style="color:#e46c0a; font-weight:bold;">diameter/corner radius</strong>." This only alters internal parameters and not the displayed tool name in the list, which can lead to extreme confusion. For such changes, please use "<strong style="color:#e46c0a; font-weight:bold;">Copy and Paste then Rename</strong>" or "<strong style="color:#e46c0a; font-weight:bold;">New Tool Creation</strong>."</li><br>    <li><strong style="color:#e46c0a; font-weight:bold;">Understand Tool Number and Compensation:</strong> If your workshop's tool management is flexible and operators manually touch off tools and input compensation, then you can set "<strong style="color:#e46c0a; font-weight:bold;">Tool Number</strong>," "<strong style="color:#e46c0a; font-weight:bold;">Compensation</strong>," and "<strong style="color:#e46c0a; font-weight:bold;">Length Compensation</strong>" to 0 in the software. However, if you need the program to automatically output these, you should set the corresponding values. Remember, these changes are only reflected in the G-code after "<strong style="color:#e46c0a; font-weight:bold;">Post-processing</strong>."</li><br>    <li><strong style="color:#e46c0a; font-weight:bold;">Master Multiple Creation Methods:</strong> Learn the two primary methods: "<strong style="color:#e46c0a; font-weight:bold;">Copy and Paste</strong>" and "<strong style="color:#e46c0a; font-weight:bold;">New Tool Creation</strong>." They are essentially the same, so choose whichever you prefer.</li><br></ul></p>
